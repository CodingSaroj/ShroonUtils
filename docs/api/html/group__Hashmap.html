Detailed Description

A hashmap (also called hashtable) implementation. This is similar to the std::unordered_map from C++ STL.

Macro Definition Documentation

◆ SUTL_HASHMAP_BUCKET_COUNT

#define SUTL_HASHMAP_BUCKET_COUNT   32

The number of buckets a SUTLHashmap will have. If it is less than or equal to 0 then it is set to 32 which is also the default value if it is not set.

More buckets will decrease search time but data will be fragmented across locations in memory. This might cause cache misses and hence slow down accessing the entries.

Less buckets will increase search time but data will be less fragmented across locations in memory. This will decrease cache misses and hence accessing the entries won't take much time.

Choose this value according to your use case.

for (i = 0; i < SUTL_HASHMAP_BUCKET_COUNT; i++)\

tv * name##_v = ((tv *)hm.Values[i] + j);\
expr\
}\
}\
}

Executes expr for every entry in hm.

Parameters

tvThe value type of hm.
hmThe hashmap to iterate.
nameThe prefix for current entry. Key will have suffix _k and value will have suffix _v.
exprThe code block to execute for every entry.
